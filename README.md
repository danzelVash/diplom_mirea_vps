# Smart Home Control System

Базовый каркас дипломного проекта для системы управления внутренней инфраструктурой жилого помещения.

## Структура

- `services/` — сервисы системы.
- `docs/` — архитектурные описания и взаимодействие сервисов.
- `deploy/` — базовые файлы для локального развертывания.

## Сервисы

- `api-gateway` — внешняя точка входа.
- `edge-bridge-service` — сетевое взаимодействие с `edge-agent`.
- `device-service` — устройства, состояния и команды.
- `context-service` — агрегированный контекст помещения.
- `scenario-service` — сценарии и принятие решений.
- `voice-service` — обработка голосовых команд.
- `vision-service` — видеоаналитика и идентификация.
- `notification-service` — уведомления и тревожные события.
- `edge-agent` — агент на `Raspberry Pi 5` для локальной и offline-работы.

## Режимы работы

- `online` — основные вычисления и координация выполняются центральными сервисами.
- `offline` — `edge-agent` продолжает базовое управление локально, а сложные функции временно ограничиваются.
